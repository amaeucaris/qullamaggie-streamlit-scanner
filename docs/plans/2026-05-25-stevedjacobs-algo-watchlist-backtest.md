# SteveDJacobs Algo Watchlist + Backtest Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Replicate the observable parts of SteveDJacobs' "Swing Trading Algo Watchlist" scanner and make it backtestable without pretending we know his proprietary formulas.

**Architecture:** Add a separate SteveAlgo module next to the existing Qullamaggie/Stockbee scanners. The module computes source-traced daily metrics, classifies symbols into visible watchlist buckets (`White Up`, `Entry`, `Yellow`, plus internal risk/status labels), and provides an event-based backtester for each bucket. The Streamlit app gets a new view, but scanner logic remains pure/testable in `qull_scanner/`.

**Tech Stack:** Python, pandas, pytest, Streamlit, yfinance-derived OHLCV cache already used by the app.

## Observable scanner spec v0

Universe gate:
- US-listed common stocks only where possible.
- Market cap >= $1B. If market cap unavailable, row is excluded from SteveAlgo v0 and marked `market_cap=N/D` in diagnostics.
- Dollar volume 20D >= configurable threshold, default $50M.
- Price >= configurable threshold, default $5.

Core metrics:
- Returns: 21D, 42D, 63D.
- EMA5, EMA10, EMA20, EMA50; SMA50, SMA100, SMA200.
- ATR10/20/50 and ATR percent.
- 52W low/high diff and range position.
- DCR = `(Close - Low) / (High - Low) * 100`.
- Gap % = `(Open - PrevClose) / PrevClose * 100`.
- Daily change % = `(Close / PrevClose - 1) * 100`.
- Open change % = `(Close / Open - 1) * 100`.
- Darvas high/low using rolling 20D high/low shifted one bar to avoid lookahead.
- Extension ATRs: `(Close - EMA10/20/50/SMA50) / ATR`.
- Relative strength rank among current universe using 63D return percentile.
- Trend strength score from MA alignment + 52W range + returns + DCR.

Watchlist buckets, transparent v0 approximation:
- `White Up`: early upward pivot; close breaks above prior 10D high or Darvas inner high, EMA10 rising, close above EMA10, DCR >= 60, not too extended over EMA10 (< 1.5 ATR10).
- `Entry`: stronger eligible condition; price >= EMA10 >= EMA20 >= EMA50, price >= SMA50 >= SMA100 >= SMA200, 21/42/63D returns positive, relative strength >= 85, trend strength >= 80, DCR >= 50, reward/risk >= 3.
- `Yellow`: caution/extended continuation; Entry-like trend strength but extension high (price-to-EMA20 >= 1.5 ATR20 or price-to-SMA50 >= 2.5 ATR50) or DCR weak while trend remains intact.

Status labels:
- `Pilot`: White Up but not Entry.
- `Buy`: Entry and extension acceptable.
- `Buy Ext`: Entry but extended.
- `Hold`: prior Entry still trend-valid; for backtest only after a position exists.
- `Avoid`: fails risk/liquidity/universe gates.

Backtest v0:
- Event entry at next open after a signal date, no same-bar execution.
- One position per ticker at a time.
- Default stop: max(Darvas low, EMA20 - 1 ATR20) or configurable.
- Default exit: stop hit intraday, close below EMA20, or max hold bars.
- Slippage/commission configurable; default conservative slippage 10 bps.
- Metrics: trades, win rate, expectancy R, profit factor, max drawdown R, OOS split, bucket contribution, best-5 contribution, random eligible-entry benchmark.
- Capital authorization remains 0%; this is research only.
